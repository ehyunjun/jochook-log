# 조축로그

조축로그는 조기축구 팀을 위한 쿼터별 포메이션 작성 및 경기 기록 웹입니다.

팀장이 경기 당일 빠르게 팀원을 등록하고, 쿼터별 포메이션을 만든 뒤, 경기 후 득점/어시스트/출전횟수 같은 기록을 간단히 남길 수 있도록 만든 가벼운 MVP입니다.

## 주요 기능

- 팀 생성
- 팀 이름과 팀원 목록을 공유 URL로 생성 및 불러오기
- DB 저장 버튼으로 팀원, 포메이션, 메모, 경기 기록까지 포함한 전체 상태 공유
- 팀원 등록 및 삭제
- 팀원별 출전횟수, 득점, 어시스트 누적 기록 확인
- 팀원 카드 클릭 시 상세 기록 및 최근 경기 기록 확인
- 포메이션 화면에서 경기 날짜와 상대팀 입력 후 경기 기록으로 저장
- 경기 준비 단계에서 출전 선수 선택 및 임시 용병 추가
- 용병 이름 수정 및 삭제
- 1~10쿼터 설정 및 쿼터별 포메이션 저장
- 쿼터 탭의 `+` 버튼으로 최대 10쿼터까지 추가
- 4-4-2, 4-3-3, 4-2-3-1 포메이션 선택
- 미배치 후보선수 목록에서 드래그 앤 드롭 또는 클릭으로 포지션 배치
- 포메이션 보드 안에서 배치된 선수 자리 이동 및 교체
- 포메이션 슬롯을 후보선수 목록으로 드래그해 배치 해제
- 모바일 화면에 맞춘 원형 포지션 슬롯과 반응형 포메이션 보드
- 포지션별 `+` 버튼으로 메모 저장
- 포메이션 보드 상단에 팀명, 쿼터, 포메이션 표시
- 저장된 경기를 선택해 득점자와 어시스트 기록
- 저장된 경기 기록 목록 표시
- 포메이션 보드와 후보선수 목록 패널을 함께 이미지로 복사

## 사용 기술

- HTML
- CSS
- JavaScript
- localStorage
- html2canvas CDN
- Supabase JS CDN

기본 기능은 별도 백엔드 없이 브라우저 `localStorage`를 중심으로 동작합니다.
이미지 복사 기능은 `html2canvas`를 CDN으로 불러와 사용합니다.

현재 버전은 서버 없이 `localStorage`를 사용합니다.
팀 코드 참가 기능 대신 팀원 정보를 URL로 공유하고 불러오는 방식을 제공합니다.
실시간 동기화는 지원하지 않으며, 공유 URL 생성 시점의 팀원 정보만 전달됩니다.

Supabase 연동 시 `DB 저장` 버튼으로 현재 팀 전체 상태를 저장할 수 있습니다.
공유 코드를 입력하면 팀원, 포메이션, 메모, 경기 기록이 포함된 상태를 불러올 수 있습니다.
현재 방식은 전체 state를 `teams.app_state` JSONB로 저장하는 MVP 방식이며, 자동 저장이나 실시간 동기화는 지원하지 않습니다.

- URL 공유: 팀 이름과 팀원 목록만 공유
- DB 공유: 팀원, 포메이션, 메모, 경기 기록까지 포함한 전체 상태 공유

포메이션 만들기 화면에서 경기 날짜와 상대팀을 입력한 뒤 저장하면 경기 기록 화면에 해당 경기가 생성됩니다.
경기 기록 화면에서는 저장된 경기를 선택하여 득점자와 어시스트를 추가할 수 있습니다.

## 실행 방법

1. 프로젝트 폴더에서 `index.html` 파일을 브라우저로 엽니다.
2. 팀을 생성합니다.
3. 팀원을 추가하고 포메이션과 경기 기록을 작성합니다.

저장 데이터는 브라우저 `localStorage`에 보관됩니다.
